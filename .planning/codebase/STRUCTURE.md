## Naming Conventions

**Files:**
- snake_case for Python files: `settings.py`, `urls.py`, `manage.py`
- All lowercase for standard Django files

**Directories:**
- PascalCase for Django project package: `JokesForProject/`
- lowercase for standard directories: `templates/`

**Special Patterns:**
- `__init__.py` - Package marker
- `wsgi.py`, `asgi.py` - Django server entry points

## Where to Add New Code

**New Django App:**
- Create via: `python manage.py startapp <app_name>`
- Location: Project root (e.g., `jokes/`)
- Register in: `JokesForProject/settings.py` INSTALLED_APPS

**New Model:**
- Implementation: `<app_name>/models.py`
- Migrations: `<app_name>/migrations/`

**New View:**
- Implementation: `<app_name>/views.py`
- Routes: `<app_name>/urls.py`, included in `JokesForProject/urls.py`

**New Template:**
- Global templates: `templates/`
- App templates: `<app_name>/templates/<app_name>/`

**Static Files:**
- Create directory: `static/`
- Or per-app: `<app_name>/static/<app_name>/`

**Tests:**
- Per-app: `<app_name>/tests.py` or `<app_name>/tests/`

## Special Directories

**.venv/:**
- Purpose: Python virtual environment
- Source: Created via `python -m venv .venv`
- Committed: No (should be in .gitignore)

**__pycache__/:**
- Purpose: Python bytecode cache
- Source: Auto-generated by Python
- Committed: No (should be in .gitignore)
